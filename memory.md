- Всегда отвечать пользователю по-русски.
- Категорически не упоминать служебное имя, бренд или происхождение агента в ветках, commit
  messages, PR title/body, issues, labels, milestones, changelog, документации, комментариях к коду,
  generated artifacts и обычных ответах пользователю. Не добавлять служебные префиксы в стиле имени
  агента.
- gh CLI в sandbox-режиме запускать только вне песочницы с `sandbox_permissions=require_escalated`;
  если escalation недоступен, не использовать gh.
- Перед GitHub CLI действиями проверять auth через `gh auth status` вне песочницы.
- После каждого `git push` в PR проверять точный `head SHA` через
  `gh pr view --json headRefOid,statusCheckRollup` и ждать checks через `gh pr checks --watch` или
  эквивалент.
- Не называть PR/CI успешными, пока GitHub не показывает `SUCCESS` для required checks на точном
  текущем `head SHA`; `queued` и `pending` называть именно так.
- Для PR локально прогонять тот же класс проверок, что CI для затронутой области; при TypeScript,
  backend bootstrap-path, env/config, test mocks или import-time коде отдельный `build`/`tsc`
  обязателен.
- Во всех `.md` файлах использовать только относительные ссылки; абсолютные файловые ссылки
  запрещены.
- Playwright: не использовать системные браузеры или реальные профили; разрешён только строго
  изолированный Playwright-managed Chrome for Testing.
- Категорически запрещено использовать `rm -rf`. Массовые удаления директорий запрещены; удалять
  файлы можно только точечно после явного перечисления путей и отдельного подтверждения
  пользователя.
